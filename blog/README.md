# 1. Semantic Versioning

### Semantic Versioning 이란?
- 소프트웨어 버전 관리의 표준화된 규칙
- [Major].[Minor].[Patch]
- Major
  - 하위 호환성을 깨뜨리는 중요한 변경이 있을 때 올림
  - 주로 기존 API의 변경 또는 기능의 큰 변화를 의미
- Minor
  - 하위 호환성을 유지하면서 기능이 추가될 때 올림
  - 주로 새로운 기능이 소개되나, 기존 코드에 큰 영향을 주지 않는 변경사항
- Patch
  - 하위 호환성을 유지하면서 기능의 버그를 수정할 때 올림
  - 새로운 기능이나 API 변경 없이 오직 버그 수정 관련된 업데이트
- ^4.3.2
  - 4.3.X 버전이 출시될 경우 자동으로 최신 버전을 다운받아 사용
  - 4.X.X 버전이 출시될 경우 자동으로 최신 버전을 다운받아 사용
  - 5.X.X 버전이 출시될 경우 최신 버전을 다운받지 않음
